# Compound Heterozygous and Digenic Parkinson's Disease Literature Review

## Overview

This repository contains the scripts used to curate, harmonize, annotate, and analyse variants reported in the literature review:

**Compound Heterozygous and Digenic Forms of Parkinson's Disease: A Systematic Review**

The repository includes two main workflows:

1. Compound heterozygous (CH) cases
2. Digenic (DI) cases

These scripts were developed as research workflows to support data curation, variant harmonization, annotation, and manuscript preparation. They are provided to ensure transparency and reproducibility of the analyses reported in the manuscript.

---

## Important Note

This repository is **not intended to function as a standalone software package** and should not be executed from start to finish as a single pipeline.

The scripts were developed iteratively during the review process and contain multiple independent analysis sections used at different stages of the project, including:

* Literature search updates
* Variant annotation
* ClinVar and HGMD matching
* CPRA harmonization
* Annovar input generation
* Dataset cleaning
* Summary statistics
* Manuscript tables and figures
* Reference indexing and supplementary material preparation

Many sections depend on intermediate files generated in previous steps, while others were used only for specific analyses reported in the manuscript.

Users interested in reproducing the study should follow the workflow section-by-section rather than attempting to run the entire script sequentially.

---

## Workflow Summary

### 1. Literature Curation

The initial datasets were assembled through systematic literature searches and manual review.

The scripts include code used to:

* Update literature searches
* Compare search iterations
* Cross-reference publications with external resources (e.g. MDSGene)
* Generate final study inclusion lists

### 2. Variant Harmonization

Reported variants were standardized across publications.

This included:

* Harmonization of variant nomenclature
* Mapping equivalent transcript representations
* Generation of canonical variant identifiers
* Construction of CPRA representations

### 3. Variant Annotation

Variants were annotated using information obtained from:

* ClinVar
* HGMD-derived reference tables
* Genoox Franklin
* Annovar

Annotations were used to:

* Classify variant consequences
* Identify benign variants
* Harmonize duplicate representations
* Generate final variant datasets

### 4. Dataset Cleaning

Patient-level datasets were filtered to remove:

* Homozygous cases
* Benign variants
* Records lacking sufficient variant information
* Variants that could not be reliably harmonized

Cleaned datasets generated by these procedures were used for all downstream analyses reported in the manuscript.

### 5. Manuscript Analyses

The scripts contain sections used to generate:

* Variant counts
* Gene-specific summaries
* Study-level summaries
* Demographic analyses
* Age-at-onset analyses
* Supplementary tables
* Manuscript reference indexes

---

## Dependencies Between Scripts

The compound heterozygous and digenic workflows are not completely independent.

Some analyses performed in the compound heterozygous workflow require files generated by the digenic workflow, particularly during manuscript preparation and reference indexing.

For this reason, users should consider the two scripts as components of a single analytical framework rather than two fully independent pipelines.

---

## External Resources

Several resources used during the study are not distributed with this repository, including:

* HGMD-derived reference tables
* Annovar databases
* Gene-specific ClinVar tables
* Genoox Franklin annotations
* Literature search exports obtained from bibliographic databases

Placeholder paths are retained in the scripts to indicate where these resources were used.

Users wishing to reproduce the analyses will need to obtain equivalent versions of these resources independently and update file paths accordingly.

---

## Reproducibility

The scripts are provided in the form used for the published analyses. While every effort has been made to document the workflow, these scripts reflect a real-world research process rather than a software package designed for automated execution.

Researchers wishing to reproduce specific results are encouraged to inspect the relevant sections of the code and execute them sequentially with the required input files.
